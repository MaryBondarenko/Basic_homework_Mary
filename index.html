Основні команди

  • git config --global user.name

    To set your Git username, run the git config –global user.name command. You should specify both your first and last name but your username can be anything you want to attach to your commits. Your Git username does not need to be the same as your version control username, such as the one you use on GitHub.

  • git config --global user.email

    Git дозволяє встановити глобальне ім’я користувача та адресу електронної пошти для кожного проекту. Ви можете встановити або змінити свій ідентифікатор git за допомогою команди git config. Зміни стосуються лише майбутніх комітів. Ім’я та електронна адреса, пов’язані з комітами, які ви зробили до внесення змін, не зачіпаються

  • git init

    Команда git init створює нове сховище Git.
    Його можна використовувати для перетворення існуючого проекту без версії в репозиторій Git або ініціалізації нового порожнього сховища. Більшість інших команд Git недоступні за межами ініціалізованого репозиторію, тому це зазвичай перша команда, яку ви запускаєте в новому проекті.

  • git status

    Команда git status використовується для відображення стану сховища та проміжної області. Це дозволяє нам бачити відстежені та невідстежувані файли та зміни. Ця команда не показуватиме жодних записів або інформації про фіксацію. Здебільшого він використовується для відображення стану між командою Git Add і Git Commit.

  • git add

    Це перша команда в ланцюжку операцій, що наказує Git «зберегти» знімок поточного стану проекту історії коммітів. Коли git add використовується як окрема команда, вона переносить очікуючі зміни з робочого каталогу розділ проіндексованих файлів

  • git log

    За умовчанням (без аргументів) git log перераховує комміти, зроблені в репозиторії у зворотному напрямку хронологічному порядку – останні коміти знаходяться вгорі

  • git commit

    git-commit - Запис змін до сховища

  • git checkout

    witch branches or restore working tree files

  • git branch

    List, create, or delete branches

  • git push

    Update remote refs along with associated objects

  • git pull

    Fetch from and integrate with another repository or a local branch

  • git merge

    Join two or more development histories together

  • git clone

    Clone a repository into a new directory

    • .gitignore файл - це gitignore потрібний для приховання файлів та папок від системи контролю версій Git.
    Зазвичай приховують файли конфігурації (особливо з паролями), тимчасові файли і папки.

    Advanced

    • git revert

      Revert some existing commits

    • git reset

      Reset current HEAD to the specified state

    • git rebase

      Reapply commits on top of another base tip

    • git cherry-pick

      Apply the changes introduced by some existing commits

    • git stash

      Stash the changes in a dirty working directory away
